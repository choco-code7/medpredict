Disease Prediction System Documentation

1. Introduction

In healthcare, predictive models for disease diagnosis and prognosis play a crucial role in early detection and effective treatment planning. The Disease Prediction System aims to address this need by providing a platform for users to input their medical data and obtain predictions for various diseases. By integrating machine learning algorithms with a user-friendly web interface, the system facilitates quick and accurate disease assessment, contributing to improved healthcare outcomes.

2. Features

  • Disease Prediction: Users can input their medical data and receive predictions for diseases such as blood pressure abnormalities, diabetes, heart disease, breast cancer, Parkinson's disease, and liver pathology.
  • Input Validation: The system validates user inputs to ensure they are within acceptable ranges and formats, providing error messages for invalid inputs.
  • Confidence Percentage: Predictions are accompanied by confidence percentages indicating the level of certainty associated with each prediction.
  • Visualization: Users can visualize their input values compared to normal ranges for each disease, aiding in interpretation and understanding.
  • Comprehensive Documentation: The system provides extensive documentation and tutorials on its functionality, architecture, and machine learning techniques employed.

3. Architecture

The Disease Prediction System follows a client-server architecture, with the backend developed in Python and the frontend built using the Streamlit framework. The key components include:

  • Backend: Developed in Python, the backend incorporates machine learning models trained on medical datasets. It handles data processing, model prediction, and communication with the frontend.
  • Frontend: Built using Streamlit, the frontend provides an interactive user interface where users can input their medical data and view predictions. It includes features for input validation, visualization, and documentation display.
  • Machine Learning Models: Trained models for each disease are stored and loaded into memory when the application starts. These models are used for making predictions based on user input data.
  • Data Preprocessing: Medical datasets undergo preprocessing to clean, standardize, and format the data, ensuring its suitability for machine learning algorithms.

4. Usage

To use the Disease Prediction System:

  1. Access the web application through the provided URL.
  2. Select a disease from the sidebar menu.
  3. Input relevant medical data into the provided fields.
  4. Click the "Predict" button to generate predictions.
  5. View the prediction results accompanied by confidence percentages.
  6. Visualize user input values compared to normal ranges for each disease.
  7. Refer to the comprehensive documentation for further information on system functionality and machine learning techniques.

5. Development Environment

To develop and deploy the Disease Prediction System, the following environment is recommended:

  • Python: Version 3.6 or higher.
  • Libraries: NumPy, Streamlit, Matplotlib, Pickle.
  • IDE: Any Python-compatible IDE (e.g., PyCharm, VSCode).
  • Operating System: Compatible with Windows, macOS, and Linux.

6. Future Enhancements

Future enhancements for the Disease Prediction System may include:

  • Improvements to user interface and visualization features for enhanced usability.
  • Expansion of disease coverage and inclusion of additional machine learning models for more comprehensive predictions.
  • Implementation of user authentication and feedback mechanisms to enhance user experience and system security.
  • Integration with external medical databases or APIs for real-time data retrieval and analysis.

7. Conclusion

The Disease Prediction System represents a valuable tool for disease diagnosis and prognosis, leveraging machine learning techniques to provide accurate predictions based on user-input medical data. With its user-friendly interface, comprehensive documentation, and potential for future enhancements, the system aims to contribute to improved healthcare outcomes and empower users to make informed decisions about their health.

Heart Dataset Documentation

Overview:

The "heart" dataset contains clinical and demographic information about patients, aiming to predict the presence of heart disease. It includes features such as age, sex, chest pain type, resting blood pressure, serum cholesterol level, fasting blood sugar level, resting electrocardiographic results, maximum heart rate achieved, exercise-induced angina, ST depression induced by exercise relative to rest, slope of the peak exercise ST segment, number of major vessels (0-3) colored by fluoroscopy, and thalassemia. The target variable, 'target', indicates the presence (1) or absence (0) of heart disease.

Dataset Information:

  • Dataset Name: Heart Dataset
  • Source: UCI Machine Learning Repository
  • Number of Instances: 1025
  • Number of Features: 14

Features:

  1. age: Age of the patient (in years).
  2. sex: Gender of the patient (0 = female, 1 = male).
  3. cp: Chest pain type (1 = typical angina, 2 = atypical angina, 3 = non-anginal pain, 4 = asymptomatic).
  4. trestbps: Resting blood pressure (in mm Hg).
  5. chol: Serum cholesterol level (in mg/dl).
  6. fbs: Fasting blood sugar > 120 mg/dl (0 = false, 1 = true).
  7. restecg: Resting electrocardiographic results (0 = normal, 1 = having ST-T wave abnormality, 2 = showing probable or definite left ventricular hypertrophy).
  8. thalach: Maximum heart rate achieved.
  9. exang: Exercise-induced angina (0 = no, 1 = yes).
  10. oldpeak: ST depression induced by exercise relative to rest.
  11. slope: Slope of the peak exercise ST segment (1 = upsloping, 2 = flat, 3 = downsloping).
  12. ca: Number of major vessels colored by fluoroscopy (0-3).
  13. thal: Thalassemia (3 = normal, 6 = fixed defect, 7 = reversible defect).
  14. target: Presence of heart disease (0 = no, 1 = yes).

Target Variable:

Target: This binary variable indicates the presence or absence of heart disease. Individuals with heart disease are labeled as 1, while individuals without heart disease are labeled as 0.

Potential Use Cases:

  1. Developing predictive models to identify individuals at risk of heart disease based on their clinical and demographic characteristics.
  2. Exploring the relationship between various risk factors and the likelihood of heart disease.
  3. Assessing the importance of different features in predicting the presence of heart disease.

Availability:

The "heart" dataset is publicly available and can be accessed from the UCI Machine Learning Repository.

Diabetes Dataset Documentation

Overview:

The "diabetes" dataset contains clinical measurements and other attributes for 768 patients, with the aim of predicting whether a patient has diabetes (the 'Outcome' variable). These attributes include information such as the number of pregnancies, glucose level, blood pressure, skin thickness, insulin level, BMI (Body Mass Index), diabetes pedigree function, and age.

Dataset Information:

  • Dataset Name: Diabetes Dataset
  • Source: UCI Machine Learning Repository
  • Number of Instances: 768
  • Number of Features: 9

Features:

  1. Pregnancies: Number of times pregnant.
  2. Glucose: Plasma glucose concentration after 2 hours in an oral glucose tolerance test.
  3. BloodPressure: Diastolic blood pressure (mm Hg).
  4. SkinThickness: Triceps skin fold thickness (mm).
  5. Insulin: 2-Hour serum insulin (mu U/ml).
  6. BMI: Body Mass Index (weight in kg/(height in m)^2).
  7. DiabetesPedigreeFunction: Diabetes pedigree function (a function which scores likelihood of diabetes based on family history).
  8. Age: Age of the patient (years).
  9. Outcome: Binary indicator of diabetes status. '0' indicates no diabetes, while '1' indicates the presence of diabetes.

Target Variable:

Outcome: This binary variable indicates the presence or absence of diabetes mellitus. Individuals with diabetes mellitus are labeled as 1, while non-diabetic individuals are labeled as 0.

Potential Use Cases:

  1. Development of predictive models to identify individuals at risk of developing diabetes based on their clinical measurements and attributes.
  2. Analysis of the relationships between different risk factors and the likelihood of diabetes.
  3. Exploration of feature importance and identification of key predictors for diabetes diagnosis.

Availability:

The "diabetes" dataset is publicly available and can be accessed from the UCI Machine Learning Repository.

Parkinsons Dataset Documentation

1. Overview

The Parkinson's Disease dataset is a collection of biomedical voice measurements obtained from individuals with and without Parkinson's disease. It comprises acoustic features extracted from sustained phonations of the vowel sound "ah" from 195 subjects, including 147 patients with Parkinson's disease (PD) and 48 healthy individuals (HC). The dataset aims to facilitate the development of predictive models for Parkinson's disease diagnosis based on voice characteristics.

Dataset Information:

  • Name: Parkinsons
  • Source: UCI Machine Learning Repository
  • Number of Instances: 195
  • Number of Features: 24 (23 features and 1 target variable)

Attribute Information:

  1. name: Identifier for each individual.
  2. MDVP:Fo(Hz): Average vocal fundamental frequency.
  3. MDVP:Fhi(Hz): Maximum vocal fundamental frequency.
  4. MDVP:Flo(Hz): Minimum vocal fundamental frequency.
  5. MDVP:Jitter(%): Variation in vocal fundamental frequency.
  6. MDVP:Jitter(Abs): Absolute jitter measure.
  7. MDVP:RAP: Variation in vocal fundamental frequency per unit of time.
  8. MDVP:PPQ: Variation in vocal fundamental frequency per unit of time.
  9. Jitter:DDP: Variation in vocal fundamental frequency per unit of time.
  10. MDVP:Shimmer: Variation in vocal amplitude.
  11. MDVP:Shimmer(dB): Variation in vocal amplitude in decibels.
  12. Shimmer:APQ3: Variation in vocal amplitude.
  13. Shimmer:APQ5: Variation in vocal amplitude.
  14. MDVP:APQ: Variation in vocal amplitude.
  15. Shimmer:DDA: Variation in vocal amplitude.
  16. NHR: Ratio of noise to tonal components in the voice.
  17. HNR: Ratio of tonal components to noise in the voice.
  18. RPDE: Nonlinear dynamical complexity measure.
  19. DFA: Signal fractal scaling exponent.
  20. spread1: Nonlinear dynamical complexity measure.
  21. spread2: Nonlinear dynamical complexity measure.
  22. D2: Nonlinear dynamical complexity measure.
  23. PPE: Nonlinear dynamical complexity measure.
  24. status: Binary indicator of Parkinson's disease presence (1 for PD, 0 for healthy).

Target Variable:

status: This binary variable indicates the presence or absence of Parkinson's disease. Individuals with Parkinson's disease are labeled as 1, while healthy individuals are labeled as 0.

Potential Use Cases:

  1. Classification models to predict the presence or absence of Parkinson's disease based on the extracted features.
  2. Regression models to predict clinical measures or severity of Parkinson's disease based on vocal characteristics.
  3. Clustering techniques to explore patterns and subgroups within the dataset, potentially identifying distinct phenotypes or subtypes of Parkinson's disease.

Dataset Availability:

The "parkinsons" dataset is publicly available and can be accessed from the UCI Machine Learning Repository.

Breast Cancer Dataset Documentation

Overview:

The "breast-cancer.csv" dataset is a collection of clinical measurements and features extracted from breast cancer biopsy samples. These features are used to predict the diagnosis of breast cancer, which can be either malignant (cancerous) or benign (non-cancerous). The dataset contains information about various attributes such as radius, texture, perimeter, area, smoothness, compactness, concavity, concave points, symmetry, and fractal dimension, among others.

Dataset Information:

  • Dataset Name: Breast Cancer Dataset
  • Source: UCI Machine Learning Repository
  • Number of Instances: 569
  • Number of Features: 32 (including the target variable 'diagnosis' and an identifier 'id')

Features:

The dataset contains the following features:

  1. id: Unique identifier for each biopsy sample.
  2. diagnosis: Diagnosis of the breast cancer (target variable). It can be either 'M' for malignant or 'B' for benign.
  3. radius_mean: Mean of distances from center to points on the perimeter.
  4. texture_mean: Standard deviation of gray-scale values.
  5. perimeter_mean: Mean size of the core tumor.
  6. area_mean: Mean area of the core tumor.
  7. smoothness_mean: Mean of local variation in radius lengths.
  8. compactness_mean: Mean of perimeter^2 / area - 1.0.
  9. concavity_mean: Mean severity of concave portions of the contour.
  10. concave points_mean: Mean number of concave portions of the contour.
  11. symmetry_mean: Mean symmetry of cell nuclei.
  12. fractal_dimension_mean: Mean of "coastline approximation" - 1.
  13. radius_se: Standard error of the mean of distances from center to points on the perimeter.
  14. texture_se: Standard error of gray-scale values.
  15. perimeter_se: Standard error of the size of the core tumor.
  16. area_se: Standard error of the area of the core tumor.
  17. smoothness_se: Standard error of local variation in radius lengths.
  18. compactness_se: Standard error of perimeter^2 / area - 1.0.
  19. concavity_se: Standard error of severity of concave portions of the contour.
  20. concave points_se: Standard error of number of concave portions of the contour.
  21. symmetry_se: Standard error of symmetry of cell nuclei.
  22. fractal_dimension_se: Standard error of "coastline approximation" - 1.
  23. radius_worst: Worst or largest mean of distances from center to points on the perimeter.
  24. texture_worst: Worst or largest standard deviation of gray-scale values.
  25. perimeter_worst: Worst or largest size of the core tumor.
  26. area_worst: Worst or largest area of the core tumor.
  27. smoothness_worst: Worst or largest local variation in radius lengths.
  28. compactness_worst: Worst or largest perimeter^2 / area - 1.0.
  29. concavity_worst: Worst or largest severity of concave portions of the contour.
  30. concave points_worst: Worst or largest number of concave portions of the contour.
  31. symmetry_worst: Worst or largest symmetry of cell nuclei.
  32. fractal_dimension_worst: Worst or largest "coastline approximation" - 1.

Target Variable:

The target variable in this dataset is 'diagnosis', which indicates the diagnosis of breast cancer. It can take two values: 'M' for malignant (cancerous) and 'B' for benign (non-cancerous).

Potential Use Cases:

  1. Classification models to predict breast cancer diagnosis based on clinical measurements and features.
  2. Feature selection techniques to identify the most influential features in determining cancer diagnosis.
  3. Understanding the relationships between different features and their impact on cancer diagnosis.

Availability:

The "breast-cancer.csv" dataset is publicly available and can be accessed from the UCI Machine Learning Repository.

Model Comparison across Multiple Datasets

In this analysis, three machine learning models, namely Logistic Regression (LR), Support Vector Machine (SVM), and Gaussian Naive Bayes (GNB), were trained on four different datasets: heart disease, diabetes, Parkinson's disease, and breast cancer. The performance of these models was evaluated using several evaluation metrics, including accuracy, precision, recall, and F1 score.

Evaluation Matrices

  • Accuracy: Measures the proportion of correctly classified instances out of the total instances.
  • Precision: Measures the proportion of true positive instances out of all predicted positive instances.
  • Recall: Measures the proportion of actual positives that were correctly predicted by the model.
  • F1 Score: Harmonic mean of precision and recall, providing a balance between precision and recall.
